Agile

Agile model can be seen as Software Development Life Cycle model in which the focus is put on process adaptability and satisfied customer (Samuel, 2020: 19). The model has recently been widely used approach in the software development process (Samuel, 2020: 21). The five main values of using agile approach, is communication, simplicity, feedback, courage and humility. The approach can be characterized as a group of software development methodologies that are based on incremental and iterative approach.

Big Bang model

Big Bang model can be characterized as very simple, because it requires little planning, a lot of programming and funds. Very small amount of planning is used in this model. It is good model to use when customer is not sure what he or she wants and what will be necessary in the future. Big Bang model is not good for big software projects, and should better be used in the learning and experimenting process (Tutorialspoint, 2020b: 4).

Iterative model

Iterative project management approach can be seen as alternative to the Waterfall model. Risks are identified early when using this model and it is possible to react in fast and efficient way. (Kruchten, 2001: 3-4). In the iterative project management approach the development process is cut in a series of repeated loops or iterations. With each iteration, new features are added or old ones are improved. At the end of each iteration, a new version of the software is produced.

Spiral model

Spiral model can be characterized as the combination of iterative and SDLC (Software Development Life Cycle) model. It can be seen as chhosing and SDLC and then combining it with the cyclic process iterative model. It is important to note that risk consideration is a part of the model. The Spiral model starts with determining objectives and constraints of the software at the start of one iteration. Then prototyping the software takes place, and it also includes risk analysis.

V - model

In the V model development and testing activity is started based on requirement specification document. Developer team work in the design and after that start implementation, testing team plans testing. Activities are happening parallel. Activities in Waterfall model and in V model are characterized as similar (STC Admin, 2012). V-model can also be called Verification and Validation model. Testing activity happens in each phase. Typical representation of the V model represents software development activities on the left side and testing phases on the right side.

Waterfall model

Gourab Dutta (Dutta, 2020: 1-3) notes that the Waterfall Model was first Process Model to be introduced. It is also referred to as a linear-sequential life cycle model. The main principle of the model is that each phase must be completed before the next phase can start. This type of software development model is basically used for small projects without uncertain requirements. In the end of each phase, there is a review and it determines if the project is on the right path.
